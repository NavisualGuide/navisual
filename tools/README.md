# AI-bbox spike — tooling

Throwaway scripts to compare three candidate locator approaches on a shared
fixture set. The authoritative spec is
[`docs/ai-bbox-spike-plan.md`](../docs/ai-bbox-spike-plan.md).

## Layout

| File | Purpose |
|---|---|
| `bbox_spike.py` | Main runner — loops over fixtures, calls every provider, scores, writes CSV. |
| `grid_draw.py` | Set-of-Marks 10×10 grid overlay for Approach B. Library + CLI. |
| `ocr_preprocess.py` | Tier 1 OCR preprocessing (2× Lanczos + contrast) for Approach C-preprocessed. |
| `_spike_metrics.py` | IoU, size-bucket, cap-height bin, grid-cell helpers. Pure functions. |
| `_spike_ocr.py` | Self-contained Windows.Media.Ocr backend + `find_text` matcher. |
| `requirements.txt` | Runtime deps (Pillow, httpx, dotenv, winrt). |
| `spike_output/` | Generated by a run — `results.csv` + `raw/*.json`. |

## Gemini SDK choice

The spike calls the Gemini REST API **directly via `httpx`**, the same pattern
we already use in `legacy/src/ai/gemini_client.py`, so no SDK import is
actually required at runtime. `google-genai` is pinned in
`requirements.txt` as the current SDK in case you want to poke at the API in
a REPL. We chose `google-genai` over the older `google-generativeai` because
Google's own docs mark the former as the go-forward SDK.

## Step-by-step

### 1. Create a venv and install deps

```bash
python -m venv tools/venv
tools/venv/Scripts/pip install -r tools/requirements.txt
```

(On non-Windows the `winrt-*` packages silently skip via
`sys_platform == "win32"` markers, but Approach C won't work there — the
spike is a Windows-only exercise.)

### 2. Capture fixtures

Follow Step 1 of `docs/ai-bbox-spike-plan.md`. In short: capture 10
screenshots at the 768×432 resolution the AI receives today (you can use the
app's own `capture_active_window` + downscale, or any other capture path),
save them as JPEGs under `fixtures/bboxes/`. One file per scene.

### 3. Write a sidecar per screenshot

For each `NN-scene.jpg`, create `NN-scene.json` alongside it. Schema and an
example live in [`../fixtures/bboxes/README.md`](../fixtures/bboxes/README.md).

### 4. API keys

Put them in the repo-root `.env` file:

```
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=AIza...
```

`python-dotenv` loads this automatically at the start of every run. **Do
not commit `.env`.**

### 5. Dry-run

Parses every sidecar and prints a summary of what *would* be run. No API
calls made.

```bash
tools/venv/Scripts/python tools/bbox_spike.py --dry-run
```

### 6. OCR-only smoke test (no API keys needed)

```bash
tools/venv/Scripts/python tools/bbox_spike.py --only-approach C
```

This exercises the full pipeline end-to-end: fixture load, OCR run (both
baseline and preprocessed), scoring, CSV write. Good for catching
environment issues before you burn AI tokens.

### 7. Full run

```bash
tools/venv/Scripts/python tools/bbox_spike.py
```

Produces:

- `tools/spike_output/results.csv` — one row per prediction.
- `tools/spike_output/raw/<fixture>__<target>__<approach>.json` — raw
  provider responses for postmortem.

A compact summary prints on stdout after the run completes.

### 8. Partial sweeps

```
--only-approach A     # Claude + Gemini raw bbox only
--only-approach B     # Claude + Gemini grid cell only
--only-approach C     # OCR baseline + preprocessed only
```

### 9. Model overrides

```
--claude-model claude-sonnet-4-6         # default
--gemini-model gemini-2.5-flash          # default
```

## Gotchas

- **Cell-label regex.** Accepts `A1`..`J10`. Anything outside that range is
  discarded. If Claude returns `K11` (model drift), the prediction is
  recorded as "not found".
- **Gemini bbox format.** Native is `[ymin, xmin, ymax, xmax]` in 0–1000.
  The runner converts to `(x, y, w, h)` pixels before scoring.
- **Claude raw bbox format.** The tool schema asks for `[x0, y0, x1, y1]`
  pixels; the runner handles swapped corners defensively.
- **Rate limits.** Each fixture × target makes up to 4 AI calls (A-Claude,
  A-Gemini, B-Claude, B-Gemini). With 10 fixtures × 3 targets, that's 120
  calls. Gemini free tier is 1500/day — fine. Anthropic has burst limits;
  if you hit 429 the exception bubbles up and the affected prediction is
  recorded as an error in `raw/*.json`.
- **OCR availability.** Approach C requires Windows + the `winrt-*`
  packages. On other OSes Approach C predictions record
  `WindowsOCRUnavailable` and move on.
- **Fixture image must exist on disk.** The sidecar's `image` field is a
  relative path resolved against the sidecar directory.
